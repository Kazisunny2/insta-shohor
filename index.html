What does javaScript event loop do?

Before we go into the event loop, let's have a look at how the task queue works. A queue is a data structure that operates on the First in First Out principle, which means that when jobs are added to the queue, they are removed in the same sequence. Tasks processed by Web APIs and sent to the Task Queue are then returned to the Call Stack to have their results written out. The event loop is a process that waits for the Call Stack to be empty before attempting to push callbacks from the Task Queue to the Call Stack. When the Stack is cleared, the event loop is triggered, and the Task Queue is checked for available callbacks. If any, it pushes it to the Call Stack, waits for the Call Stack to clear, and then repeats the procedure.
